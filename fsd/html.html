Introduction

HTML
HyperText Markup Language (HTML), Cascading Style Sheets (CSS), and JavaScript are the languages that run the web.

HTML defines the content of every web page on the Internet. By “marking up” your raw content with HTML tags, you’re able to tell web browsers how you want different parts of your content to be displayed.

Development Setup

The basic workflow for web developers is to write HTML code in their text editor and save as using a .html file extension and view those changes in a web browser.

Element / Tags

HTML 'Element' also known as 'Tag' are reserved keyword with an opening <> and closing </> pair of angle bracket's surrounding the content which helps the browser read and render the content.

Example below
<tag> Content here </tag>

Structure of a Webpage

Below is a simple HTML skeleton required for a webpage

<!DOCTYPE html>
Tell browsers that this is an HTML5 document

<html>
Element is the root element of an HTML page
All webpage content are wrapped around a <html> tag anything inside will be rendered by browser. Html element is further divided into two main elements Head and Body.

<head>
Contains all things that are required to render the page but you don’t necessarily want the user to see, like Metadata (used by google, etc), Title (displayed on top of browser), css stylesheet (used for modifying appearance of the webpage)

  • <title>
    One of the most important pieces of metadata is the title of your web page, Browsers display this in the tab for your page, and Google displays it in search engine results.

<body>
The bulk of our HTML markup will live in the <body> element, which represents the visible content of the webpage.

Block level vs Inline Elements

Based on purpose of the element, it can be either block or inline elements. We can make block elements inline and vice versa using CSS Display style property.

  • Block Level Elements

    Block level elements takes use up all of the remaining line for itself, even if not used. Like <h1> element will span a line irrespective of the width.
    Block level elements always starts on a new line.

  • Inline Elements

    Leaves the remaining line for other inline elements to be used. Two inline elements will wrap up side by side.
    Block level never uses inline elements spaces until specified.

Generic & Self Closing Elements

Depending on their usage some of HTML elements can be "generic" or "self-closing / empty".

  • Generic Elements

    There are Two important generic elements

    • <div>

      <div> considered as building block, lego block of html helps in layout and structuring the HTML.
      <div> is generally used to group a bunch of elements together to struture, style them for presentation purpose. it is also know as wrapper tag
      <div> is a Block Level Element.

    • <span>

      <span> element is similar to <div> element generally used to wrap content of text instead of elements, to give them specific styling for presentation.
      <span> is a Inline Element.

    Generic doesnot define it purpose, they are also know as wrapper tag.

  • Self Closing Elements / Empty Elements

    Self closing element close themselves and are not accompanied by a closing tag.
    Line breaks <br/> and Horizontal rules <hr> are the most common self closing elements you’ll find.
    The trailing slash (/) in all Self Closing HTML elements is entirely optional.
    Note the lack of / in the <hr> tags

    Self closing elements are also know as "Empty Elements"

Attributes

HTML attributes are key value pair [key="value"] used by Elements, that provide additional functionality to the elements.
<button key="value"> content </button>

Some Common attributes are

  • ID
  • Class
  • Style

ID

ID is an attribute that helps uniquely identify a specific elements, often used for CSS styling, Manipulating Data, DOM with javascript / jQuery.
ID is represented using '#' symbol while using CSS/Javascript
<button id="formSubmit"> Login </button>

CSS styling using ID
#formSubmit {
  border: 1px solid dodgerblue;
  padding: 4px 8px;
}

DOM Manipulation using jQuery
$("#formSubmit").on("click", function(){
  border: 1px solid red;
});

Class

Class is an attribute that helps us group similiar or dis-similar elements together.
Class is represented using '.' symbol while using CSS/Javascript
<button class="primary"> Login </button>
<p class="primary"> Lorem ipsum is a placeholder text commonly used to demonstrate the visual form of a document </p>

CSS styling using Class
.primary {
  font-size: 16px;
  color: blue;

Lists

List element is made of two element wrapper tag for list like <ul> , <ol> & individual list item (<li>) used in combination to display a simple list of items

There are two type of list

  • Unordered List
  • Ordered List

Below image illustrate the difference between both list, we can change the list display style using style attribute list-style-type: type;

  • Ordered List

    Ordered list as the name suggest displays a list in an ordered way where sequence is important.
    this list is used to display "steps to follow in a tutorials"  , "requirements", etc

    <ol> stands for ordered list used a wrapper tag, individual items inside of <ol> is wrapped using <li> element

    <ol>
       <li> list one </li>
       <li> list two </li>
    </ol>
    Results in the following output

    1. list one
    2. list two

  • Unordered List

    Just like the name suggest in unordered list the sequence of the list items doesnot matter. the list is displayed using bullet points instead of numbering system.

    <ul>
       <li> list one </li>
       <li> list one </li>
    </ul>
    Results in the following output

    • list one
    • list two
